# プロジェクト概要: nokoriごはん

## プロジェクトの目的
「残り物を入力するだけで、簡単にレシピを作成できるアプリ」

## 技術スタック
- **フロントエンド**: Next.js 15 (App Router)
- **バックエンド**: Ruby on Rails 8 (API専用)
- **データベース**: PostgreSQL
- **AI連携**: OpenAI GPT-5 mini API
- **認証**: JWT認証
- **対応デバイス**: PC・スマートフォン（レスポンシブデザイン）

## 主要機能
- ユーザー認証とユーザー管理
- ユーザー設定管理（調理人数、アレルギー、道具、調味料、栄養目標）
- 食材在庫管理（数量・賞味期限付き）
- 2パターンレシピ生成（残り物活用 + 栄養最適化）
- 買い出しリスト機能
- レシピ保存機能

## データ構造
- **Users**: 認証情報
- **UserPreferences**: アレルギー、道具、調味料、栄養目標、デフォルト人数
- **Ingredients**: 食材マスター
- **UserIngredients**: ユーザーの食材在庫（数量付き）
- **Recipes**: 生成・保存されたレシピ（パターン識別、人数情報付き）
- **ShoppingLists**: 買い出しリスト（人数対応）

## 開発状況
- Ruby 3.4.3
- Rails 8
- 実装済み機能: 基本的なAPI群、認証、レシピ生成機能
- テスト: RSpec設定済み（ファクトリ含む）
